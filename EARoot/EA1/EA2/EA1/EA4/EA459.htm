D_ITEM_CONDITION_REFERENCE_CODE_KEY Public bigint
Notes: It is a numerical value foreign key to DIM_REFERENCE_CODE.. Specifies the condition of the Lift/Escalator corresponding to the event that has occured. Examples of condition are Trapping, Not Operational, Awaiting Restart, Not in Service, Action Required etc.<br/>
D_ITEM_CONDITION_PRIORITY_REFERENCE_CODE_KEY Public bigint
Notes: It is a numerical value foreign key to DIM_REFERENCE_CODE. Defines the priority (a numerical value), to the condition of a Lift/Escalator. The lower the number the higher the priority of the condition. It is possible that a Lift/Escalator has mutiple conditions at the same time. For ex. if a Lift has a trapping event (condition Trapping) and a not operational event (condition Not Operational). The conditionPriority (number) for trapping is lower, meaning a higher priority and this is the condition that should be displayed in report/display board etc. Example values. 30/40/60/70 etc.<br/>
D_RESTORE_ITEM_EVENT_KEY Public bigint
Notes: It is a numerical value foreign key to DIM_ITEM_EVENT. This specifies the Key to the (placeholder) Event Id of a future restore event,  for the current Event record. Once the restore event occurs, it indicates that the current (fault) event has been restored / rectified. <br/>
D_ITEM_EVENT_TYPE_REFERENCE_CODE_KEY Public bigint
Notes: It is a numerical value foreign key to DIM_REFERENCE_CODE. Every event is assigned one of the 3 types viz, "Event" meaning it is a doublesided event, "reference" mening a single sided event, "Out of Service" meaning a double sided out of service event.<br/>
