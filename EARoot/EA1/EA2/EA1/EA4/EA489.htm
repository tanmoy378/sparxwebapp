PAYMENT_APPLICATION_NAME Public varchar(80)
Notes: The name/code assigned to the payment application. Starts with PA- followed by digits.<br/><br/>A Payment application comes into picture once a Work Order is marked complete. This is where the payments made to contractors/artisan teams gets processed. When a work order is marked complete, it gets submitted to Payment application (this payment application module sits within Citadel Salesforce).<br/>Once the work order is submitted to Payment Application, the payment application is reviewed, approved/ rejected within Salesforce<br/>Actual physical money is paid outside of Salesforce<br/>Each work order is linked to only one payment application at a point of time. There could be multiple work orders pointing to same Payment application<br/><br/>
PAYMENT_APPLICATION_STATUS Public varchar(255)
Notes: Status of the payment applicatio. Values like - New, Submitted to NR Commercial Manager, Submitted to Contractor for Review, Approved, Rejected<br/><br/>
PRELIMS Public numeric(18,2)
Notes: Most contractors get paid for work order based on the work done for that specific work order. However, in some cases, few contractors are paid a fixed amount every month/time period - irrespective of how many work orders they have during that month or not. This fixed value paid to contractor is called prelim value. Prelim values are calculated only when a set of work orders worked on by a contractor in a month are marked as Work Complete. That's when the work orders get submitted to payment application and a Prelim value is entered here by a contractor. Prelim can be - Fixed Prelim (where a fixed amount is entered) ; Percentage prelim (where a percentage is applied to the submitted cost. Prelim concept is required for both PPM and Reactive Work orders<br/><br/>
FIXED_PRELIMS Public numeric(18,2)
Notes: Fixed amount entered for a contractor in Payment application - irrespective of the number of work <br/><br/>
PRELIM_SUM Public numeric(18,2)
Notes: Total of all prelim values - Fixed and percentage<br/><br/>
